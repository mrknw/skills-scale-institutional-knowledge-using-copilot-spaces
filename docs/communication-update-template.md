## Template Usage Guidelines

### For Communications Leads
- **Customize**: Adapt templates to project context, but maintain core structure
- **Consistency**: Use the same template format throughout a project
- **Timing**: Send updates at predictable cadences (e.g., every Friday)
- **Brevity**: Keep updates concise - busy stakeholders appreciate clarity
- **Action-oriented**: Clearly state what decisions or actions are needed
- **Audience**: Tailor technical detail to audience (executives vs. engineers)

### For All Team Members
- **Request templates**: Ask Communications Lead for appropriate template
- **Provide input**: Share updates with Communications Lead for compilation
- **Review**: Communications Lead should review with PM before sending
- **Archive**: Store all communications in project documentation for reference
- **Feedback**: Share suggestions for template improvements
